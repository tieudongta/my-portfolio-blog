To-Do List Tutorial Step 7

Posted on May 03, 2025

Tutorial Description

Let's continue building your own To-Do List App from step 6.In this step, we'll implement the feature that allows you to Inline Edit Tasks- this will let users click and edit a task name directly in the list.

Goals

  1. Enable users to edit the task text inline, with changes saved automatically.

Step 7-1: Update Task Rendering in renderTasks()

In your <li> rendering section, update the task display to make the text editable:


<span contenteditable="true" 
      onblur="editTask(${index}, this)" 
      onkeydown="checkEnter(event)">
  ${task.text}
</span>

This allows:

  • contenteditable="true": enables direct editing.
  • onblur: saves changes when user clicks away.
  • onkeydown: handles pressing Enter to confirm edit.

Step 7-2: Add the editTask() Function

In your script.js, add:


function editTask(index, span) {
  const newText = span.textContent.trim();
  if (newText) {
    tasks[index].text = newText;
    saveAndRender();
  } else {
    // Optional: delete task if text is emptied
    deleteTask(index);
  }
}

Step 7-3: Add checkEnter() to Handle Enter Key

This makes the edit feel natural:


function checkEnter(event) {
  if (event.key === 'Enter') {
    event.preventDefault();
    event.target.blur(); // triggers onblur/editTask
  }
}

Step 7-4: Style Editing Behavior in CSS

To highlight when a task is being edited:


li span[contenteditable="true"]:focus {
  outline: 2px dashed #007bff;
  background-color: #eef;
}

Done! You can now:

  • Click task text to edit.
  • Press Enter or click away to save.
  • Clear the task to delete it (optional behavior)
