Mastering Regular Expressions: A Practical Journey

Posted on May 25, 2025

Regular Expressions (regex) are a powerful tool for working with text, enabling pattern matching, validation, and extraction. Whether you're validating user input, scraping data, or parsing logs, regex is an essential skill for every developer. In this post, I'll walk you through a structured 5-day regex learning path I followed—starting from basics to advanced concepts, each reinforced with hands-on practice. We'll wrap it up with a real-world “boss level” mini project that ties everything together.

  1. Day 1: The Basics

    • Exact matches: /abc/
    • Anchors: ^ (start), $ (end)
    • Character classes: \d (digit), \w (word)
    • Quantifiers: * (0 or more), + (1 or more), {n,m} (range)

    Practice

    • Match zip codes, usernames, and emails.

  2. Day 2: Grouping & Alternation

    • Groups: () for capturing submatches
    • Alternation: | to match one of many patterns
    • Non-capturing groups: (?:...)

    Practice

    • Match phone numbers with or without area codes.
    • Extract first and last names from full names.

  3. Day 3: Character Sets and Negation

    • Sets: [aeiou] to match vowels
    • Ranges: [a-z], [0-9]
    • Negation: [^aeiou] matches non-vowels

    Practice

    • Match all uppercase letters
    • Match all non-digit characters

  4. Day 4: Special Characters and Escaping

    • Escape sequences: \\, \\., \\*
    • Special sequences: \\b (word boundary), \\s (space), \\S (non-space)

    Practice

    • Match file extensions like .jpg, .png
    • Detect punctuation

  5. Day 5: Assertions

    • Lookaheads: (?=...), (?!...)
    • Lookbehinds: (?=<...), (?<!...)
    • Named groups: (?...)
    • Backreferences: \\1, \\k

    Practice

    • Match prices only when followed by “USD”
    • Find usernames not preceded by “@”
    • Match repeated words

  6. Final Boss-Level Project

    To solidify all this knowledge, I built a resume parser using only vanilla JavaScript and regex.

    Features

    • Paste raw resume text
    • Extracts:
      • Name
      • Email
      • Phone (123-4567 format)
      • DOB (YYYY-MM-DD)
      • GitHub & LinkedIn URLs
      • Skill list (comma-separated)
    • Displays parsed data in JSON format

    Regex Highlights Used

    • Anchors (^, $)
    • Word characters (\\w), digits (\\d)
    • Quantifiers ({3,}, +, *)
    • Lookaheads and optional groups
    • Escaped characters and grouping
