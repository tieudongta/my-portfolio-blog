Day 2: Intermediate Patters on Regex Learning Path

Posted on May 19, 2025

Great job on completing Day1 of your regex learning! You've already covered some essential basics. For Day 2, we can build on that foundation with more intermediate concepts that will help you write more powerful and flexible expressions.

  1. Character Sets([abc])

    Meaning:

    Matches any one character between the brackets.

    /[aeiou]/.test("hat"); // true (continas 'a')
    "cat".match(/[cmt]/g); // ['c', 't']

    Practice

    • Match any vowel: /[aeiou]/
    • Match the digits 1, 2, or 3: /[123]/
    • Match "a", "b", or "c" in "abc123"

  2. [^abc] - Negated Character Set

    Meaning:

    Matches any character that is NOT in the set

    /[^0-9]/.test("a"); // true (not a digit)
    "abc123!".match(/[^a-zA-Z0-9]/g); // ['!']

    Practice

    • Match anything except digits: /[^0-9]/
    • Match non-vowels in "regex": /[^aeiou]/g

  3. . - Dot (Wildcard)

    Meaning:

    Matches any character except newline.

    /.at/.test("cat"); // true
    "flat bat mat".match(/.at/g); // ['lat', 'bat', 'mat']

    Practice:

    • Match any 3-letter word ending in "at": /.at/
    • Match the second character in "a_b": /a.b/

  4. | Alternation

    Meaning:

    Acts like OR between alternatives.

    /dog|cat/.test("hotdog"); // true
    "catdog".match(/cat|dog/g); // ['cat', 'dog']

    Practice

    • Match "yes" or "no": /yes|no/
    • Match "blue" or "green" in a string

  5. () - Grouping/Capturing

    Meaning

    Groups patterns and captures what was matched.

    /(abc)+/.test("abcabc"); // true
    "ha ha ha".match(/(ha)+/); // ['ha']

    Practice

    • Match one or more "ha": /(ha)+/
    • Match full names like "John Smith": /([A-Z][a-z]+) ([A-Z](a-z)+)/

  6. \s - Whitespace

    Meaning:

    Matches spaces, tabs, newlines.

    /\s/.test("hello world"); // true
    "hello world".match(/\s+/); //[' ']

    Practice:

    • Match a space: /\s/
    • Match multiple spaces: /\s+

  7. \S - None-whitespace

    Meaning:

    Matches any non-space characer.

    /\S/.test(" "); // false
    "abc 123".match(/\S+/g); //['abc', '123']

    Practice:

    • Match non-space sequences: /\S+/

Summary Table

Pattern Meaning Example Match
[abc] One of a, b, or c "a"
[^abc] Not a, b, or c "x"
. Any character (except newline) "c", "1", "!"
| Alternation (OR) /cat|dog/
() Capturing group "(abc)"
(?:) Non-capturing group "abc"
\s Whitespace " " (space)
\S Non-whitespace "a","1", "!"

Practice Challenges

  1. Match the words "red", "green", or "blue".

    /red|green|blue/

  2. Match a U.S. phone number format like (123) 456-7890:

    /\(d{3}\)\s\d{3}-\d{4}/

  3. Match all words (separated by whitespace):

    /\s+/g

  4. Match three-letter words ending in "at":

    /\b.at\b/g

  5. Match "ha" or "haha" or "hahaha" using grouping:

    /(ha)+/

Real World Practice

You'll use what you learned to extract structured info from messy text.

Data

const messyData = `
User: Mike_007
Email: mike007@example.com
Note: Happy customer.

User: sam99
Email: not-an-email@wrong
Comment: Ordered twice.

User: Alice1990
Email: alice_1990@sample.net
Message: Hello there!
`;

Your Tasks

  1. Extract all valid emails (word characters only before @, then word chars, then .com/.net/ect):

    /\b\w+@\w+\.\w{2,3}\b/g

  2. Extract all usernames:

    /User:\s(\w+)/g

  3. Extract all lines with a comment or note:

    /^(Note|Comment):.+$/gm

Day 2 Quiz

  1. What does the patter /[a-z]/ match?

    1. Any lowercase letter
    2. A digit from 0 to 9
    3. Any uppercase letter
    4. Any character except a-z

    Explanation:
    [a-z] is a character class that matches any single lowercase letter from a to z.

  2. What does /[^0-9]/

    1. A number
    2. Only whitespace
    3. Any character not a digit
    4. Only lowercase letters

    Explanation
    [^0-9] is a negated character class that matches any single character except digits 0 through 9.

  3. What does /\s+/ match in a string?

    1. One or more spaces
    2. One or more non-whitespace characters
    3. Any single word
    4. A line break only

    Explain:
    \s matches any whitespace character (spaces, tabs, line breaks), and + means one or more occurrences.

  4. Which regex matches either "cat" or "dog"

    1. /cat|dog/
    2. /cat&dog/
    3. /cat||dog/
    4. /cat\/dog/

    Explain:
    The pipe | acts as an OR operator matching either "cat" or "dog".

  5. What will /(ha)+/.test("hahaha") return?

    1. false
    2. true
    3. undefined
    4. "ha"

    Explain:
    The regex matches one or more repetitions of "ha", and "hahaha" is "ha" repeated 3 times, so .test() returns true.

  6. Which of the following will match a word character followed by a space?

    1. /\w\s/
    2. /\s\w/
    3. /\w\w/
    4. /\S\s/

    Explain
    \w matches any word character (letter, digit, underscore), followed by \s which matches a whitespace character.

  7. Which regex matches a username that starts with a letter followed by digits (like "a123")

    1. /^\d+\w$/
    2. /^[a-zA-Z]\d+$/
    3. /^[0-9][a-z]+$/
    4. /^\w+$/

    Explain:
    ^[a-zA-Z] means the string starts with a letter
    \d+ means one or more digits
    $ means the string ends here
    So the entire string must be a letter followed by digits.

  8. What does /\S+/g return when used on "Hi there!"?

    1. ["Hi there!"]
    2. ["Hi", "there!"]
    3. [" "]
    4. ["Hi", " ", "there!"]

    Explain:
    \S+ matches one or more non-whitespace characters. The global flag g finds all such matches separately.

  9. Which regex matches the three-letter words "cat", "bat", or "rat"?

    1. /[cbr]at/
    2. /cat|bat|rat/
    3. Both A and B
    4. /.*at/

    Explain:
    [cbr]at matches any of 'c', 'b', or 'r' followed by "at"
    cat|bat|rat explicitly matches any of those three words
    Both will match the same set of words.

  10. What does /(?:abc)+/.test("abcabc") return?

    1. false
    2. true
    3. "abc"
    4. undefined

    Explain:
    (?:abc)+ matches one or more repetitions of the sequence "abc". The test() returns true because "abcabc" is "abc" twice.
