Day 4 - Groups, Alternation & Captures

Posted on May 21, 2025

Great job on completing Day 3 of your regex learning! Today's focus is on Groups, Alternation, and Capture Groups, continuing in the same clear and persistent format. Let's go!

  1. () - Grouping

    Meaning:

    Groups multiple tokens together - works like parantheses in math. Helpful for repeating, capturing, or alternating entire subpatterns.

    /(ha)+/.test("hahaha"); // true
    "haha".match(/(ha)+/); // ['haha']

    Practice

    • Match repeated "ha": /(ha)+/
    • Match "hello" or "world": /(hello|world)/

  2. | - Alternation

    Meaning:

    Acts like a logical OR. Matches either the expression on the left or right.

    /cat|dog/.test("hotdog"); // true
    /yes|no/.test("maybe"); // false

    Practice

    • Match "apple" or "banana": /apple|banana
    • Match "Mr" or "Mrs": /Mr|Mrs/

  3. () - Capturing Groups

    Meaning:

    Stores the matched subpattern inside parentheses for retrieval via .match() , .replace() , etc.

    const match = /(\d{4})-(\d{2})-(\d{2}).exec("2025-05-19");
    //match[1] = "2025", match[2] = "05", match[3] = "19"

    Practice:

    • Extract year, month, day from date: /(\d{4})-(\d{2})-(\d{2})/
    • Capture area code from phone: /\((\d{3}))\)/

  4. (?:) - Non-Capturing Groups

    Meaning:

    Groups tokens without saving the match in a capture group.

    /(?:ha)+/.exec("hahaha"); // ['hahaha'], no captured group

    Practice

    • Match repeated "ha" without capturing: /(?:ha)+/
    • Group "Mr" or "Mrs" but only capture name: /(?:Mr|Mrs)\. (\w+)/

  5. \1, \2, etc - Backreferences

    Meaning

    Refers back to a previousl captured group. Useful for repeated patterns or symmetry.

    /(\w)\1/.test("book"); // true ('oo')

    Practice

    • Match any double letter. /(\w)\1/
    • Match repeated word: /\b(\w+)\s+\1\b/

Summary Table

Pattern Meaning Example Match
(abc) Capturing group "abc"
(?:abc) Non-capturing group "abc" but not stored
cat|dog Alternation (OR) "cat"
(\w)\1 Backreference to Group 1 "letter", "book"
(\d+)-(\d+) Captures two groups of digits "2025-19" -> ["2025", "19"]

Practice Challenges

  1. Extract the first and last name from "John Smith"

    /(\w+)\s+(\w+)/

  2. Match HTML-like tags (e.g., <b>text</b>)

    /\<(\w+)\>.*\<\/\1\>/

  3. Match repeated words (e.g., "yes yes")

    /\b(\w+)\s+\1\b/

  4. Extract dates like 2023-12-31

    /(\d{4})-(\d{2})-(\d{2})/

  5. Match double characters (e.g., "tt" in "letter"):

    /(\w)\1/

Day 4 Quiz

  1. What does the regex /(ha)+/ match?

    1. Exactly one "ha"
    2. One or more repetitions of "ha"
    3. Only "haha"
    4. "h" followed by multiple "a"s

    Explanation:
    (ha)+ means match "ha" at least once, so "ha", "haha", "hahaha", etc

  2. Which regex matches either "yes" or "no"?

    1. /yes||no/
    2. /yes|no/
    3. /yes no/
    4. /(yes)(no)/

    Explanation
    The pipe | means "OR" - this matches "yes" or "no"

  3. What does (\d{3})-(\d{4}) capture from 123-4567

    1. ["123-4567","4567"]
    2. ["123", "4567"]
    3. ["4567"]
    4. Nothing - it doesn't match.

    Explanation:
    Group 1: \d{3} -> "123"
    Group 2: \d{4} -> "4567"

  4. What does a non-capturing group look like?

    1. /(abc)/
    2. /abc?/
    3. (?:abc)
    4. (?abc)

    Explanation:
    The ?: tells the engine: group the pattern, but don't save the match.

  5. What does the regex /(\w)\1/ match?

    1. Two identical digits
    2. Two identical characters next to each other
    3. Two any characters
    4. A single character

    Explanation:
    (\w) captures one character, \1 looks for the same character repeated.

  6. Which regex matches names like "Mr. Smith" and "Mrs. Jones" but only captures the name?

    1. /Mr|Mrs\.(\w+)/
    2. /(?:Mr|Mrs)\. (\w+)/
    3. /(Mr|Mrs)\. (\w+)/
    4. /Mr|Mrs|(\w+)/

    Explanation:
    (?:Mr|Mrs) groups without capturing, while (\w+) captures the name.

  7. What is the purpose of \1 in a regex?

    1. Escape character
    2. Captures one character
    3. Backreference to the first capture group
    4. Repeat the regex once

    Explanation:
    Refers back to the first captured match - useful for detecting repeats or symmetry.

  8. Which regex would match any repeated word (e.g., "go go")?

    1. /(\w)\1/
    2. /(\w+)\s+\1/
    3. /(\w+)\1/
    4. /\b(\w+)\b/

    Explanation:
    (\w+) captures a word, \1 checks if it's repeated after space(s).

  9. What does this regex do" /<(\w+)>.*<\/\1>/

    1. Matches angle brackets
    2. Matches any HTML-like tag with matching open and close
    3. Matches any tag starting with "<"
    4. Invalid regex

    Explain:
    Captures a tag name (\w+) and uses \1 to match the closing tag correctly.

  10. What will "banana".match(/(an)+/) return?

    1. ["ana"]
    2. ["anan"]
    3. ["an","an"]
    4. ["anan", "an"]

    Explain:
    (an)+ matches one or more "an" -> it matches "anan" from the middle of "banana".
