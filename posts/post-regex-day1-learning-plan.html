Day 1: /abc/, ^, $, \d, \w, *, +, {n,m}

Posted on May 18, 2025

We'll go through concepts, examples, and practice challenges for each.

  1. /abc/ - Exact Match

    Meaning:

    Matches the exact sequence abc in a string.

    const str = "syzabc123";
    const result = /abc/.test(str); //true

    Practice

    • Does abc exist in "xyzabc"
    • Find "dog" in "hotdog"

  2. ^-Start of String

    Meaning

    Matches if the pattern starts at the beginning of the string.

    /^Hello/.test("Hello world");// true
    /^Hello/.test("Say Hello"); //false

    Practice

    • Check if s string starts with "cat".
    • Match usernames that start with a letter: /^[a-zA-Z]/

  3. $ - End of String

    Meaning

    Matches if the pattern ends at the end of the string.

    /world$/.test("Hello world"); // true
    /world$/.test("world peace"); // false

  4. \d - Digit

    Meaning:

    Matches any digit (0-9)

    /\d/.test("abc3def");//true

    "Price: 123".match(/\d+/g); //{'123'}

    Practice:

    • Match 4-digit years: /\d{4}/
    • Match 2-digit numbers

  5. \w - Word Character

    Meaning:

    Matches any letter, digit, or underscore [a-zA-Z0-9_]

    /\w/.test("_"); // true
    /\w/.test("-"); // false

    "This_is_good".match(/\w+/g); // ["This_is_good"]

    Practice

    • Match usernames: /^\w+$/
    • Match a variable name like my_var1

  6. * - 0 or More

    Meaning

    Matches 0 or more of the preceding character

    /a*/.test(""); //true
    "a".match(/a*/); //['a']
    "aaab".match(/a*/); // ['aaa']

    Practice

    • Match "ha" repeated any number of times: /ha*/
    • Match an optional "s": /cat(s)*/

  7. + - 1 or More

    Meaning

    Matches at least one of the preceding character

    /a+/.test(""); // false
    "aaa".match(/a+/); // ['aaa']

    Practice

    • Match one or more digits: /\d+/
    • Match one or more word characters: /\w+/

  8. {n,m} - Repetition Quantifier

    Meaning:

    Matches between n and m repetitions of the preceding pattern

    /\d{2,4}.test("199"); // true (3 digits)
    "12345".match(/\d{2,4}/); // ['1234']

    Practice

    • Match 5-digit zip codes: /\d{5}/
    • Match words with 3-6 characters: /\w{3,6}/

Summary Table

Pattern Meaning Example Match
/abc/ exact string "abc"
^ start of string "^Hi" matches "Hi there"
$ end of string "Bye$ matches "Say Bye"
\d digit "4", "123"
\w word character "a", "Z", "9", "_"
* 0 or more "aaa" ,""
+ 1 or more "a", "aa"
{n,m} between n and m times "abc if n=1, m=3

Practice Challenges

  1. Match phone numbers like 123-4567

    /^\d{3}-\d{4}$/

  2. Validate a 4-digit PIN

    \^/d{4}$\

  3. Match lowercase letters only

    /^[a-z]+$/

  4. Match an email-like pattern

    /^\w+@\w+.\w{2,3}$/

Real World Challenge

You've been given a block of unstructured user data. Your task is to:

  1. Extract valid email address
  2. Find valid 4-digit PINs
  3. Check if names start with a capital letter
  4. Match any line that ends with a period
  5. Find phone numbers in 123-456 format

Input (Exmaple Data).

const text = `
John Smith
Email: john_smith23@gmail.com
PIN: 1234
Phone: 123-456

Invalid:
- Email: not-an-email@wrong
- PIN: 12
- Name: alice brown
- Phone: 1234-5678

Jane Doe
Email: jane.doe@example.com
PIN: 9876
Phone: 999-999.
`;

Your Tasks

  1. Extract all valid email address

    Must contain only word characters before @, then a domain and extension of 2-3 letters

    /\b\w+[\w.]*@\w+\.\w{2,3}\b/g
  2. Extract all 4-digit PINs /\b\d{4}\b/g
  3. Find lines where the name starts with a capital letter /^[A-Z][a-z]+[A-Z][a-z]+/gm
  4. Find lines that end with a period /^.*\.$/gm
  5. Extract all valid phone numbers in 123-456 format. /\b\d{3}-\d{3}\b/g
