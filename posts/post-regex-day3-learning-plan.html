Day 3 - RegEx Anchors, Boundaries, Lookaheads, and More Patters

Posted on May 20, 2025

Great job on completing Day 2 of your regex learning! You've already covered some immediate knowledges on Regex. For Day 3, we explore Regex Anchors, Boundaries, Lookaheads, and More.

  1. \b - Word Boundary

    Meaning:

    Matches the position between a word character and a non-word character. It doesn't consume characters -- it's just a position.

    /\bcat/.test("a cat"); // true
    /\bcat/.test("scatter"); // false

    Practice

    • Match the word "cat" only when it's a whole word: /\bcat\b/
    • Match 3-letter words: /\b\w{3}\b/g

  2. \B - Non-Word Boundary

    Meaning:

    Matches where \b doesn't, i.e., inside a word.

    /\Bend/.test("bend"); // true
    /\Bcat/.test("scat"); // true

    Practice

    • Match "end" only inside words: /\Bend/
    • Match "ing" not at the start or end: /\Bing\B/

  3. (?=...) - Positive Lookahead

    Meaning:

    Asserts that what follows the current position must match the pattern inside

    /\d(?=px)/.text("5px"); // true
    "5px".match(/\d(?=px)/); // ['5']

    Practice:

    • Match digit before "px": /\d(?=px)/
    • Match "http" only if followed by "s": /http(?=s)/

  4. (?!...) - Negative Lookahead

    Meaning:

    Asserts that what follows must NOT match the pattern.

    /\d(?!px)/.test("5em"); // true
    "5em".match(/\d(?!px)/); //['5']

    Practice

    • Match digit not before "px": /\d(?!px)/
    • Match "foo" not followed by "bar": /foo(?!bar)/

  5. (?<=...) - Positive Lookbehind (Limited browser support)

    Meaning

    Asserts that what comes before the current position must match.

    /(?<=\$)\d+/.test("$100"); // true
    "$100".match(/(?<=\$)\d+/); // ['100']

    Practice

    • Match digits after a $ sign: /(?<=\$)\d+/
    • Match word after "@": /(?<=@)\w+/

  6. (?<!...) - Negative Lookbehind (Limited browser support)

    Meaning:

    Asserts that what comes before must NOT match.

    /(?<!\$)\d+/.test("100"); // true
    "100".match(/(?<!\$)\d+/); //['100']

    Practice:

    • Match digits not after $: /(?<!\$)\d+/

Summary Table

Practice Challenges

  1. Match "cat" as a full word, not part of "scatter"

    /\bcat\b/

  2. Match any number before "px":

    /\d+(?=px)/

  3. Match any number not before "px":

    /\d+(?!px)/

  4. Extract the domain name from emails like "john@example.com"

    /(?<=@)\w+/

  5. Match words that start and end with the same letter:

    /\b(\w)\w*\1\b/g

Real World Practice

You'll use what you learned to solve real world task.

const css = `

Your Tasks

  1. Extract only the numbers used for px values:

    /\d+(?=px)/g

  2. Extract only valid domain names from emails

    /@(\w+\.\w{2,3})/g

  3. Match lines that contain a color hex code

    /^.*#{3,6}.*$/gm

Day 3 Quiz

  1. What does the regex /\bcat\b/ match?

    1. Only "cat" when it is a full word
    2. Any word containing "cat"
    3. "cat" at the start of a string
    4. "cat" anywhere, including inside other words

    Explanation:
    \b ensures it's bounded by non-word characters - not part of another word like "scatter".

  2. Which regex matches a number only if it's followed by "px"?

    1. /px\d+/
    2. /\d(?=px)/
    3. /\d(?<=px)/
    4. /px(?=\d)/

    Explanation
    This is a positive lookahead - it matches a digit only if it's flollowed by "px";

  3. What is the difference between \b and \B?

    1. \b matches inside words; \B matches outside words
    2. \b matches whole words; \B matches digits
    3. \b is a word boundary; \B is a non-word boundary
    4. They are aliases of each other

    Explanation:
    \b matches between word and non-word characters. \B does the opposite.

  4. What does /foo(?!bar)/ match?

    1. "foobar" only
    2. "foo" only if not followed by "bar"
    3. "bar" after "foo"
    4. "foo" only if it's at the end of the string

    Explanation:
    This is a negative lookahead: match "foo" but only if "bar" does not follow.

  5. Which regex matches digits only if they come after a dollar sign $

    1. /\$[0-9]+/
    2. /\d+(?<=\$)/
    3. /(?<=\$)\d+/
    4. /\d(?=\$)/

    Explanation:
    This is a positive lookbehind - the digits must be preceded by a $. Note: Escaping $ is required!

  6. What will "abc123px".match(/\d+(?=px)/) return?

    1. ["123px"]
    2. ['px']
    3. ['123']
    4. null

    Explanation:
    It matches the digits before "px" - the px isn't included.

  7. What does /\Bing/ match in "singing"

    1. "ing" at the end only
    2. Nothing = it requires a word boundary
    3. "ing" when it's inside a word
    4. The full word "singing"

    Explanation:
    \B requires no word boundary. so "ing" in the middle of "singing" qualifies.

  8. Which expression finds three-letter words only?

    1. /\b\w{3}\b/g
    2. /\w{3}/g
    3. /\b{3}/g
    4. /\B\w{3}\B/g

    Explanation:
    This ensures exactly 3 characters, bounded by word boundaries = full word.

  9. What does this match: /(?≶!\$)\d+/ ?

    1. Digits preceded by $
    2. Digits not preceded by $
    3. Digits followed by $
    4. Only the $ character

    Explain:
    This is a negative lookbehind - excludes numbers that come after $.

  10. True or False: Lookaheads and Lookbehinds consume characters.?

    1. True
    2. False

    Explain:
    These assertions don't consume characters; they just check positions.
