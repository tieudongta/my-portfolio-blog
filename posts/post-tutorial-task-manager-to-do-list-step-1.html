To-Do List Tutorial Step 1

Posted on May 03, 2025

Tutorial Description

Let's build your own To-Do List App from scratch, one step at a time - no shortcuts - so you can understand very layer (HTML -> css -> JavaScript) and why we do each part

Step 1-1: Set Up Basic HTML Structure

Let's start with a basic HTML file that defines the structure of the app: title, input field, and a list

Here's the code for index.html


<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
  <title>To-Do List</title>
  <link rel="stylesheet" href="style.css" />
</head>
<body>
  <div class="container">
    <div class="header">
      <h1>My To-Do List</h1>
      <button id="theme-toggle">Dark Mode</button>
    </div>

    <form id="todo-form">
      <input type="text" id="task-input" placeholder="Enter a new task" required />
      <button type="submit">Add</button>
    </form>

    <ul id="task-list"></ul>
  </div>

  <script src="script.js"></script>
</body>
</html>

What you've done in this step:

  • Created a container for the app
  • Added a heading and a theme toggle button
  • Added a form for new tasks
  • Left a <ul> place holder where tasks will appear
  • Linked a stylessheet (style.css) and a JavaScript file (script.js)

Now Your Turn

  1. Create a folder name to-do
  2. Create index.html in the folder
  3. Type the code above into index.html
  4. Make sure it's loading in your browser with nothing broken

Step 1-2: Add Basic CSS Styling

We'll style the layout to make it clean, centered, and user-friendly. Create a new file called style.css in the same folder as your HTML.


/* Global styles */
body {
  font-family: sans-serif;
  background-color: #f4f4f4;
  padding: 2rem;
  margin: 0;
}

/* Container styling */
.container {
  max-width: 400px;
  margin: auto;
  background: white;
  padding: 1.5rem;
  border-radius: 10px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

/* Header layout */
.header {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* Form styling */
form {
  display: flex;
  gap: 0.5rem;
  margin-top: 1rem;
}

input[type="text"] {
  flex: 1;
  padding: 0.5rem;
}

button {
  padding: 0.5rem 1rem;
  cursor: pointer;
}

/* Task list */
ul {
  list-style: none;
  padding: 0;
  margin-top: 1rem;
}

li {
  background: #eee;
  margin-bottom: 0.5rem;
  padding: 0.6rem;
  border-radius: 5px;
  display: flex;
  justify-content: space-between;
  align-items: center;
}

li.completed {
  text-decoration: line-through;
  color: #888;
}

What you've learned:

  • Layout styling (.container, .header,form)
  • How to use Flexbox for layout
  • Visual styling for each task (li)
  • Class-based styling (.completed) for completed tasks

Step 1-3: JavaScript for Task Creation

In this step, we'll:

  1. Capture the input from the user
  2. Store the tasks in the browser's localStorage to persist data.
  3. Display the tasks dynamically in the <ul> element

Sample code

Create a new file called script.js and add the following code:


// Get references to HTML elements
const form = document.getElementById('todo-form');
const taskInput = document.getElementById('task-input');
const taskList = document.getElementById('task-list');

// Initialize an array to store tasks (use localStorage for persistence)
let tasks = JSON.parse(localStorage.getItem('tasks')) || [];

// Render tasks to the UI
function renderTasks() {
  // Clear the current list
  taskList.innerHTML = '';

  // Loop through tasks and create 
  •  for each
  tasks.forEach((task, index) => {
    const li = document.createElement('li');
    li.textContent = task.text;

    // Add a 'completed' class if the task is marked as completed
    if (task.completed) {
      li.classList.add('completed');
    }

    // Append the 
  •  to the task list
    taskList.appendChild(li);
  });
}

// Add task to the array and re-render the list
function addTask(e) {
  e.preventDefault();  // Prevent form submission

  const text = taskInput.value.trim();

  if (text === '') return;  // Don't add empty tasks

  // Add the new task
  tasks.push({ text, completed: false });

  // Save tasks to localStorage
  localStorage.setItem('tasks', JSON.stringify(tasks));

  // Clear the input field and re-render tasks
  taskInput.value = '';
  renderTasks();
}

// Event listener for form submission
form.addEventListener('submit', addTask);

// Initial rendering of tasks on page load
renderTasks();

    • What's happening in the code

    • Capturing input: We get a reference to the task input field using getElementById.
    • Storing tasks: Tasks are stored in an array(tasks). They are saved to the browser's localStorage so they persist even when the page reloads.
    • Rendering tasks: The renderTasks() function is responsible for rendering the tasks as <li> elements inside the <ul> with the id task-list
    • Handling form submission: The addTask() function is triggered when the form is submitted. It adds the new task to the array, updates localStorage, and re-renders the list.

    3. Reload the page.
